# W-01 — Экраны waiter PWA (4 шт.)

Роутинг:

```
/login                          → LoginPage
/                               → TablesPage  (default)
/menu/:tableId                  → MenuPage    (новый заказ для стола)
/menu/:tableId/order/:orderId   → MenuPage    (добавить позиции к существующему)
/order/:orderId                 → OrderPage
```

Все экраны обёрнуты в `<LanGuard>` (см. W-02). Если `LanGuard` не пускает — рендерится stand-alone страница «Не в сети ресторана».

## 1. LoginPage `/login`

```
┌─────────────────────────────────────┐
│            RestOS Waiter            │
│         Введите PIN официанта       │
│                                     │
│         ●  ●  ○  ○  ○  ○            │
│                                     │
│         ┌────┬────┬────┐            │
│         │ 1  │ 2  │ 3  │            │
│         ├────┼────┼────┤            │
│         │ 4  │ 5  │ 6  │            │
│         ├────┼────┼────┤            │
│         │ 7  │ 8  │ 9  │            │
│         ├────┼────┼────┤            │
│         │    │ 0  │ ← │            │
│         └────┴────┴────┘            │
│                                     │
│         [ Неверный PIN ]            │
└─────────────────────────────────────┘
```

- Вход — **только по PIN официанта** (4–6 цифр), как на кассирском POS. Логин/пароль для waiter отключены.
- `POST /auth/waiter/pin/` с `{pin}`. На 200 — возвращает `{access, refresh, user}` (тот же контракт, что `/auth/login/`); сохранить токены в `restos.waiter.auth`, редирект на `/`.
- Авто-submit при наборе 4 цифр; можно добрать до 6. Кнопка `←` стирает последнюю цифру.
- На 401 → красная плашка «Неверный PIN», очистка введённых цифр.
- Lockout — пять неверных попыток подряд блокируют пользователя на `pin_lock_timeout_min` минут (общий механизм с касс. PIN).
- Backend: эндпоинт ищет пользователя с `role=waiter` и `is_active=true` в текущем `restaurant`. Кассирский `/auth/pin/` для waiter недоступен и наоборот.

```tsx
const { mutate, isPending } = useMutation({
  mutationFn: (pin: string) => api.auth.loginByPin(pin),
  onSuccess: ({ access, refresh, user }) => {
    authStore.setTokens(access, refresh);
    authStore.setUser(user);
    navigate("/");
  },
});
```

```
┌────────────────────────────────────────────────────┐
│ Карим · Выйти                              ● Online│
├────────────────────────────────────────────────────┤
│ [ Зал 1 ] [ Зал 2 ] [ Терраса ]                    │
├────────────────────────────────────────────────────┤
│  ┌────┐  ┌────┐  ┌────┐  ┌────┐                    │
│  │ 1  │  │ 2  │  │ 3  │  │ 4  │                    │
│  │FREE│  │OCCU│  │BILL│  │FREE│                    │
│  │ -  │  │95.0│  │45.0│  │ -  │                    │
│  └────┘  └────┘  └────┘  └────┘                    │
└────────────────────────────────────────────────────┘
```

- Зоны как табы.
- Стол — `<TableCard>`, цвет фона по статусу (free=серый, occupied=оранжевый, bill_requested=красный + мигание).
- Стол показывает: номер, сумму активного заказа (если занят), официанта (если другой — мелким шрифтом).
- **Клик free** → `<GuestsDialog>` (1–8 гостей) → после OK: `POST /tables/{id}/open/`, затем `navigate("/menu/{id}")`. Открытие стола не создаёт заказ — только помечает стол `occupied` с этим официантом. Заказ создастся при первой отправке.
- **Клик occupied (свой)** → `navigate("/order/{currentOrderId}")`.
- **Клик occupied (чужой)** → toast «Стол ведёт {имя другого официанта}» (без перехода).

```tsx
// Только обычный fetch — обновления приходят через SSE и инвалидируют кеш.
const { data: tables } = useQuery({
  queryKey: ["tables"],
  queryFn: () => api.tables.list({ zone: zoneId }),
});
// Глобально установлен useEventStream() — он сделает
// queryClient.invalidateQueries({ queryKey: ["tables"] }) при table.updated.
```

## 3. MenuPage `/menu/:tableId(/order/:orderId)?`

```
┌──────────────────────────────────────────────┐
│ ←  Стол 5 · 3 гостя                          │
├────────────┬─────────────────────────────────┤
│ Категории  │  Блюда                          │
│ • Холодные │  ┌──────┐ ┌──────┐ ┌──────┐    │
│   Горячие  │  │ 🥗   │ │ 🍚   │ │ 🍞   │    │
│   Напитки  │  │Салат │ │ Плов │ │Лепёш│    │
│   Десерты  │  │ 30.0 │ │ 45.0 │ │ 8.0 │    │
│            │  └──────┘ └──────┘ └──────┘    │
│            │   …                             │
├────────────┴─────────────────────────────────┤
│ 🛒 Корзина (4 поз. · 98.00) [ Отправить ]   │  ← sticky bottom
└──────────────────────────────────────────────┘
```

- Слева sticky-список категорий (вертикальный).
- Справа сетка `<DishTile>` (192×192). Тап → +1 в корзину; long-press → диалог «количество».
- Если блюдо `is_available=false` — карточка серая, тап заблокирован, бейдж «Стоп».
- Корзина внизу — кликабельная плашка, показывает кол-во и сумму. Клик → `<CartDrawer>`.

### CartDrawer

```
┌──────────────────────────────────────────────┐
│ Корзина                                      │
├──────────────────────────────────────────────┤
│ Плов          [-] 2 [+]    90.00  [×]       │
│ Чай           [-] 1 [+]     8.00  [×]       │
│                                              │
│                          ИТОГО:  98.00 TJS   │
├──────────────────────────────────────────────┤
│              [    Отправить    ]            │
└──────────────────────────────────────────────┘
```

- Кнопка `Отправить`:
  - Новый заказ: `POST /orders/` с `{table_id, guests_count, items}` + `Idempotency-Key`.
  - Добавление к существующему (`/menu/:tableId/order/:orderId`): `POST /orders/{orderId}/add_items/`.
- На сетевую ошибку — toast «Нет связи, повтор» и retry с тем **же** `Idempotency-Key` через 3 с.
- На 200/201 — очистить корзину (и draft), `navigate("/order/{orderId}")`.

### Drafts

`store/drafts.ts` (см. W-03) автоматически сохраняет cart на каждый change в `localStorage["restos.waiter.drafts"][tableId]`. При следующем заходе на `/menu/:tableId` восстанавливается.

После успешной отправки — draft этого стола удаляется.

## 4. OrderPage `/order/:orderId`

```
┌──────────────────────────────────────────────┐
│ ←  Заказ № 42 · Стол 5                       │
│ Статус: НОВЫЙ                                │
├──────────────────────────────────────────────┤
│ Плов                          2 × 45.00 = 90 │
│ Чай                           1 × 8.00  =  8 │
│ ────────────────────────────────────────────│
│ ИТОГО:                              98.00 TJS│
├──────────────────────────────────────────────┤
│ [ + Добавить блюда ]   [ ⏎ Запросить счёт ]  │
└──────────────────────────────────────────────┘
```

- `useQuery({ queryKey:["order", orderId] })`. Refetch триггерит SSE-хук на событиях `order.updated` для этого id (см. W-02).
- **«+ Добавить блюда»** → `navigate("/menu/{tableId}/order/{orderId}")`.
- **«Запросить счёт»** → confirm-dialog → `POST /orders/{id}/request_bill/`.
- После `bill_requested` кнопка «Запросить счёт» скрывается, появляется текст «Кассир принимает оплату».
- Когда заказ становится `done` или `cancelled` (приходит SSE `order.updated`) — auto-redirect на `/` через 3 с.

### Отмена позиции

В режиме `status=new` рядом с каждой позицией — кнопка `[×]`. Тап → `<ReasonDialog>` (4 пресета: «Гость передумал», «Ошибка кухни», «Качество», «Другое»). Отправка `POST /orders/{id}/cancel_item/  {item_id, reason}`.

### Отмена всего заказа

В нижней части — текстовая ссылка «Отменить весь заказ» (мелким шрифтом). Доступна только в `status=new`. Отправляет `POST /orders/{id}/cancel/  {reason}`.

## Глобальные элементы

### Шапка

Фиксированная сверху:
- Имя официанта слева
- Индикатор `Online / Offline` справа (отслеживает axios-ошибки за последние 10 с)
- Кнопка «Выйти» (logout: чистим store + localStorage)

### Toasts

- `toast.error(msg)` — на сетевые ошибки и 4xx с user-facing message.
- `toast.success(msg)` — на отправку заказа, запрос счёта.
- Используем `sonner` или встроенный из shadcn.

### Confirm-диалоги

Для действий без отката: «Запросить счёт», «Отменить заказ». Modal с двумя кнопками.

## Адаптивность

- Минимальная ширина — 768 px (планшет в landscape). На phone-portrait категории сворачиваются в верхний `<select>`, корзина — в нижний sheet.
- Шрифт основных кнопок ≥ 18 px, тач-таргет ≥ 48×48.
