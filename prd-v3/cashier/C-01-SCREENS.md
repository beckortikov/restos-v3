# C-01 — Экраны кассира (5 шт.)

Все экраны — `QWidget`, переключаются через `QStackedWidget` в `main.py`. Каждый экран знает только свои сигналы и зависимости из `state.py`.

```
QMainWindow
  └─ QStackedWidget
       ├─ PinLoginScreen
       ├─ TablesScreen        (после login)
       ├─ ActiveOrdersScreen  (отдельная вкладка nav)
       ├─ OrderDetailScreen   (модальное окно над TablesScreen)
       ├─ PaymentScreen       (модальное окно)
       └─ ReceiptStatusScreen (модальное окно)
```

Левая боковая панель навигации (`QToolBar` в виде вертикального бара): «Карта зала», «Активные заказы», «Выйти».

---

## 1. PIN Login

Один экран, один экран. Никаких заголовков и логотипов — только функция.

```
┌─────────────────────────────────────────┐
│                                         │
│              ┌─────────────┐            │
│              │ • • • •     │            │
│              └─────────────┘            │
│                                         │
│           ┌───┐ ┌───┐ ┌───┐             │
│           │ 1 │ │ 2 │ │ 3 │             │
│           ├───┤ ├───┤ ├───┤             │
│           │ 4 │ │ 5 │ │ 6 │             │
│           ├───┤ ├───┤ ├───┤             │
│           │ 7 │ │ 8 │ │ 9 │             │
│           ├───┤ ├───┤ ├───┤             │
│           │ ← │ │ 0 │ │ ✓ │             │
│           └───┘ └───┘ └───┘             │
│                                         │
│           [ Неверный PIN ]              │  ← красным, появляется после ошибки
└─────────────────────────────────────────┘
```

- Поле PIN — readonly `QLineEdit`, заполняется кружочками. Длина 4–6 символов.
- Numpad — `widgets/numpad.py`, кнопки 80×80.
- При нажатии ✓ → `POST /auth/pin/`. На успех — сохранить `session_token` в `keyring` и перейти на TablesScreen. На ошибку — вибро-анимация поля, показ «Неверный PIN», очистка.
- 5 неуспешных подряд → backend выдаст `AUTH_INVALID_PIN` ещё 15 минут — UI показывает «Учётка заблокирована до HH:MM».

- Каждый стол — `widgets/order_card.py` в режиме `table`. Размер ≥ 120×120, цвет фона по статусу.

## 4. OrderDetailScreen + PaymentScreen

Открывается как модальное окно (`QDialog`) над текущим экраном.

```
┌───────────────────────────────────────────────┐
│ Заказ № 42 · Стол 5 · Карим · 3 гостя         │
├───────────────────────────────────────────────┤
│ Плов                          2 × 45.00 = 90  │
│ Чай                           1 × 8.00  =  8  │
│ ─────────────────────────────────────────────│
│ ИТОГО:                              98.00 TJS │
├───────────────────────────────────────────────┤
│ ┌──────────┐  ┌──────────┐  ┌──────────────┐ │
│ │Наличные  │  │  Карта   │  │   Перевод    │ │
│ └──────────┘  └──────────┘  └──────────────┘ │
│                                               │
│        [ Оплатить и напечатать чек ]         │
└───────────────────────────────────────────────┘
```

- Кнопки методов оплаты — `QPushButton` в `QButtonGroup`. Default — «Наличные».
- При клике «Оплатить» →
  - `Idempotency-Key: uuid4()`
  - `POST /orders/42/close/` с `{payment_method: "cash"}`
  - кнопка блокируется (предотвращаем двойной клик), даже если пользователь успеет — backend всё равно вернёт тот же ответ.
- При успехе → переключение на `ReceiptStatusScreen`.
- При `409 ORDER_ALREADY_CLOSED` (например, кассир дважды нажал «Оплатить» с разных машин — теоретически) → показать toast «Заказ уже оплачен», вернуться на TablesScreen.

## 5. ReceiptStatusScreen

```
┌─────────────────────────────────────┐
│            ⏳                       │   ← spinner
│    Печать чека № 42 …               │
│                                     │
│    [ Отмена ]    [ В заказы ]       │
└─────────────────────────────────────┘
```

- Показывается сразу после успешного `close` (поле `print_job.id` пришло в ответе).
- Polling каждую 1 секунду на `GET /printing/jobs/{id}/`.
- Состояния:
  - `pending` / `printing` — спиннер.
  - `done` — иконка ✓, текст «Готово», авто-закрытие через 3 с.
  - `failed` (не финальное) — иконка !, текст «Сбой, повтор через N сек», кнопка «Повторить» (POST `/jobs/{id}/retry/`).
  - `dead` — иконка ✕, текст «Принтер недоступен после 5 попыток», кнопка «Повторить вручную» (POST `/retry/` обнуляет retries).
- Кнопка «В заказы» возвращает на `TablesScreen` независимо от статуса печати — заказ всё равно закрыт.

## Сигналы и навигация

```python
# pos/main.py (упрощённо)

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.stack = QStackedWidget()
        self.setCentralWidget(self.stack)

        self.pin = PinLoginScreen()
        self.tables = TablesScreen()
        self.orders = ActiveOrdersScreen()

        self.pin.logged_in.connect(self.on_logged_in)
        self.tables.table_clicked.connect(self.show_order_detail)
        self.orders.order_clicked.connect(self.show_order_detail)

        self.stack.addWidget(self.pin)
        self.stack.addWidget(self.tables)
        self.stack.addWidget(self.orders)
        self.stack.setCurrentWidget(self.pin)

    def on_logged_in(self):
        state.start_stream()                 # запускает SseClient (см. C-02)
        self.stack.setCurrentWidget(self.tables)

    def show_order_detail(self, order_id: int):
        dlg = OrderDetailScreen(order_id, self)
        dlg.paid.connect(lambda job_id: ReceiptStatusScreen(job_id, self).exec())
        dlg.exec()
```

## Авто-блокировка

`QApplication.installEventFilter` → таймер reset на любых input-событиях. По истечении `pin_lock_timeout_min`:

```python
state.stop_stream()
self.stack.setCurrentWidget(self.pin)
```

`session_token` остаётся в keyring; при следующем вводе PIN backend либо реюзает (если ещё валиден), либо выдаёт новый.

## (post-MVP) Подключение планшета

Минимальный хелпер-диалог для онбординга планшета официанта в Wi-Fi ресторана. Открывается с PIN Login-экрана по text-кнопке «📱 Подключить планшет» в левом верхнем углу.

```
┌─────────────────────────────────────────┐
│       Подключение планшета              │
│   Наведите камеру планшета на QR-код    │
│                                         │
│         ████  ██  ████  ██              │
│         ██ █████ ██ ████                │
│         ███ ███ █████  ██     (QR)      │
│         ██  ██ █████ ███                │
│         ████  █████ ██                  │
│                                         │
│         http://192.168.1.10/            │
│                                         │
│             [   Закрыть   ]             │
└─────────────────────────────────────────┘
```

- URL: `pos.config.get_pair_url()` → `RESTOS_PAIR_URL` если задан, иначе `http://<lan-ip>/` (определяется через socket-трюк в `pos/lib/net.py:detect_lan_ip`).
- QR рендерится через `segno` (SVG, error="h" — устойчиво к бликам на экране), отображается через `QSvgWidget`. Реализация — `pos/screens/tablet_pairing_screen.py:TabletPairingDialog`.
- **Post-MVP**: фрейма в `design/pos_cashier.pen` для этого экрана нет, диалог сделан минималистично. В Phase 2 нужно нарисовать его в Pencil и переделать 1:1.
- **Backend не задействован**: aутентификация официанта — обычная waiter-PIN (`POST /auth/waiter/pin/`). Никаких per-device токенов/моделей.
